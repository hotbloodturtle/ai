다음 이름으로 새 프로젝트를 시작해주세요: $ARGUMENTS

작업 순서:
1. 프로젝트 목적을 확인 (사용자에게 질문)
2. 적합한 기술 스택 제안
3. 폴더 구조를 먼저 보여주고 확인받기
4. 확인 후 파일 생성:
   - 기본 폴더 구조
   - README.md (한국어)
   - .gitignore (적절한 제외 목록)
   - 기본 설정 파일 (package.json 등)
   - .claude/CLAUDE.md (프로젝트 규칙)
5. git init 및 첫 커밋
6. 다음 작업 순서 안내
