$ARGUMENTS

리팩토링 규칙:
1. 먼저 현재 코드를 읽고 구조 파악
2. 리팩토링 계획을 먼저 보여주고 확인받기
3. 기존 기능은 100% 유지 (동작이 바뀌면 안 됨)
4. 변경 사항:
   - 중복 코드 제거
   - 함수/변수 이름 명확하게
   - 긴 함수는 작은 단위로 분리
   - 불필요한 복잡성 제거
   - 타입 안전성 강화
5. 변경 전/후 비교 설명
6. 테스트가 있다면 실행하여 기능 유지 확인
7. 외부 라이브러리 추가 없이 진행
